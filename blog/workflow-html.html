Flujo de trabajo con Git y GitHub

  1. Preparación del entorno

    2. Primero debes configurar tu identidad para que tus acciones queden registradas. Utiliza el comando git config para establecer tu nombre y email. Después, usa git init para inicializar el repositorio en tu carpeta.

  2. Manejo de repositorios remotos

    3. Para trabajar con repositorios en GitHub necesitas conectar tu repo local con uno remoto.
    Puedes usar git clone para obtener una copia completa de un proyecto existente, o git remote add para vincular tu repo local con uno remoto.

  3. Ciclo básico

    4. El flujo de trabajo esencial consiste en modificar archivos, revisarlos con git status, añadirlos al área de preparación con git add y luego confirmar los cambios con git commit. Este ciclo se repite mientras desarrollas.

  4. Manejo de ramas

    5. Las ramas son fundamentales para el desarrollo en paralelo. Con git branch puedes ver y crear ramas. Para moverte entre ramas usa git checkout o git switch. Crear una nueva rama y moverte a ella es tan sencillo como usar git checkout -b nueva-rama.

  5. Sincronización con GitHub

    6. Para sincronizar con el repositorio remoto, usa git fetch para ver cambios remotos antes de aplicarlos. Con git pull puedes descargar y aplicar esos cambios. Para subir tus propios cambios usa git push.

  6. Integración de código

    7. Para unir diferentes ramas de desarrollo usamos git merge. Este proceso puede generar conflictos si se modificaron las mismas líneas en ambas ramas. En ese caso, deberás resolver manualmente los conflictos y luego hacer un commit de la integración.
